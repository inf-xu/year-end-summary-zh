# 去 AI 味的年终总结（`year-end-summary-zh`）

基于零散工作素材生成更像“真人写的”年终总结：把流水账整理成管理者可快速评估的成果表述，并支持绩效评估、晋升准备、调薪沟通等场景。

## 你会得到什么

- 三档输出：精简版 / 常规版（默认）/ 详尽版
- 管理者视角：强调结果、影响范围、责任边界与证据
- 去 AI 味：减少模板化套话，增加第一人称反思与具体场景
- 信息补全：素材不足时，优先级排序追问不超过 5 个问题
- 事实边界：不编造数据；缺少量化时会标注“（需确认）/（建议补充具体数据）”

## 适用 / 不适用

**适用：**
- 素材分散（周报/月报/事项清单/碎片笔记）但需要交付正式年终总结
- 总结要提交给直属领导/管理层/HR，用于绩效或晋升讨论

**不适用：**
- 学术/课程/学生总结
- 纯个人复盘（不需要对外提交）
- 创业者/自由职业者（缺少“上级评估”语境）
- 技术方案/项目文档（不是个人工作总结）
- 英文总结（本 Skill 面向中文职场语境）

## 安装

将 `year-end-summary-zh/` 放到你的技能目录即可（不同环境路径可能不同）：

```bash
cp -R year-end-summary-zh "$CODEX_HOME/skills/"
# 或者：cp -R year-end-summary-zh ~/.codex/skills/
```

## 快速使用

把你的素材直接粘贴给助手，并说明输出版本与背景信息即可：

```text
请用 year-end-summary-zh 写一份【常规版】年终总结（800-1200 字）。

我的岗位/级别：XX
团队/业务：XX
今年目标/OKR：XX
素材（可能很碎）：
- ...
- ...
要求：不虚构数据；如果信息不够请先问我 3-5 个关键问题。
```

## 输入建议（越完整越好）

- 你的岗位、职责边界、协作对象（你主导/执行/协作是什么）
- 关键项目/事件：背景、你做了什么、怎么做的、结果/影响
- 量化指标：时间、数量、比例、对比（前后变化）或区间
- 定性证据：上级/同事/客户反馈、复盘结论、上线时间点
- 明年方向：1-3 个可执行目标或聚焦领域

## 输出规格

- **精简版**：150-250 字；1 句年度定位 + 3-5 条核心成果（每条 1 句）
- **常规版（默认）**：800-1200 字；年度概述 + 3-5 项关键成果 + 能力成长 + 不足与改进 + 明年计划
- **详尽版**：1500-2000 字；常规版 + 更完整的证据/数据/决策背景（更适合晋升备档）

## 工作流程（内部逻辑）

1. 扫描素材：识别时间线、项目、重复项与成果线索
2. 信息补全：素材不足时追问（最多 5 个问题）
3. 成果提炼：把“做了什么”翻译成“带来什么影响”
4. 去 AI 味改写：删除高频套话，增加第一人称反思与具体场景
5. 结构化输出：先结论后支撑；每段首句可独立表达关键点
6. 质量自检：直接性/节奏/真实性/精炼度（总分 40，目标 ≥ 28）
